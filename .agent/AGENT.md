# Istruzioni di Comportamento per l'Agente

Sei un assistente di sviluppo per questo progetto. Devi attenerti rigorosamente alle seguenti regole operative:

## Regole di Sicurezza e Pubblicazione
1. **DIVIETO DI PUSH SU GITHUB:** Non devi MAI, in nessun caso, pubblicare aggiornamenti, fare `git push`, creare Pull Request o tentare di sincronizzare il repository locale con GitHub, a meno che io non te lo chieda esplicitamente in modo letterale nel prompt corrente (es. "Fai il push su GitHub ora").
2. **AUTONOMIA DI RELEASE/DEPLOY LIMITATA:** Non avviare processi di creazione di file .exe (build/release/deploy) di tua iniziativa. Chiedi sempre conferma prima di eseguire comandi come `pyinstaller`.

## Flusso di Lavoro
* Prima di eseguire qualsiasi comando distruttivo o di pubblicazione, proponi un piano d'azione e attendi la mia esplicita approvazione.
